Detailed Description

Queue tests are coupled
Testing one Queue method requires calling other Queue methods. Lib Queue design intent is that data in a Queue_s is only accessible through the Queue methods. The Queue_s is intended to be an abstract data type, but it is exposed in Queue.h to make inline possible in the application's translation unit.

Empty tests are for generating function doc strings

  • Empty tests contain one line: TEST_PASS();
  • Tests start as empty.
  • If the test is useful, write it using the necessary Queue methods when they are available.
